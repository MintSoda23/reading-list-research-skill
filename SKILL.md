# Reading List Research

## Overview

Use official course reading lists as a reality check for whether a program fits the user. Social posts and rankings are noisy; reading lists reveal what students will actually read, which theories/methods the program values, and whether the user's curiosity matches the curriculum.

## Core Workflow

1. Clarify the user's target:
   - Degree level: undergraduate, master's, PhD, certificate, short course.
   - Field and subfields: e.g. design, design education, speculative design, STS.
   - Schools/programs already under consideration.
   - Personal filters: career goal, theoretical taste, methods, countries, language, budget, application timeline.

2. Find official reading-list systems and course pages:
   - Search queries:
     - `<school name> reading list <field>`
     - `<school name> <department> reading list`
     - `<school name> <program name> reading list`
     - `<school name> <course/module/studio name> bibliography`
     - `<school name> <program name> syllabus`
   - Common platform clues: Talis Aspire, Leganto, Library Reading Lists, Resource Lists, Course Reserves, module catalogue, programme specification.
   - Prefer official university domains, library systems, course catalogues, programme pages, module pages, and PDFs.

3. Pair reading lists with program descriptions:
   - Open the official program page and identify degree level, tracks, studios, modules, concentrations, required/elective structure, and learning outcomes.
   - Map reading-list titles to those modules/studios/tracks.
   - Exclude mismatched levels or goals: BA when the user wants MA, PhD-only content when the user wants taught master's, unrelated departments, outdated archived pages unless no current source exists.

4. Classify the readings:
   - Treat "Required", "Essential", "Core", "Recommended" as first-priority evidence.
   - Treat "Further", "Optional", "Additional", "Suggested" as second-priority exploration.
   - Preserve internal headings such as methods, theory, history, practice, STS, critical/speculative design, education, policy, data, etc.
   - Note repeated authors, books, journals, and themes.

5. Compare across schools when multiple programs are available:
   - Find overlaps: readings/authors that appear in several schools.
   - Identify differentiators: unique modules, unusual authors, practice orientation, theoretical orientation, technical orientation.
   - Infer fit cautiously: say "suggests" or "appears" when reading-list evidence implies a program's emphasis.
   - Do not over-rank from reading lists alone; combine with program description and user goals.

6. Build a practical reading plan:
   - Start with shared/core readings across target schools.
   - Add one or two program-specific clusters that match the user's strongest interest.
   - Keep it small: prioritize 5-12 items unless the user asks for a full bibliography.
   - Mark "read first", "sample/skim", and "optional later".

## Output Format

When answering, produce a concise research report:

- **Verdict**: 2-4 sentences on whether the program(s) fit the user's stated interests.
- **Evidence Map**: table with Program, Official source, Matched module/studio/list, Level, Priority readings, Main themes.
- **Best Signals**: strongest evidence for fit, including repeated authors/themes and core modules.
- **Mismatch Signals**: readings, modules, or level mismatches that suggest the program may not fit.
- **Starter Reading List**: prioritized items with a short reason for each.
- **Next Checks**: specific remaining sources to verify, such as current year list, module page, faculty profile, open day recording, or alumni work.

Always include source links when web research is used. Clearly distinguish sourced facts from inference.

## Search Tips

- Use the school name in English and the department name as written on the official site.
- Search both broad field terms and program-specific terms.
- Try platform names directly: `site:rl.talis.com <school> <field>`, `site:talisaspire.com <school> <field>`, `site:leganto.* <school> <field>`.
- If a list page exposes a department/category breadcrumb, open it to see all lists in the same department.
- If a reading-list platform blocks indexing, search the university library site for "reading lists" and browse by department.
- If no list is public, use module catalogues, syllabi PDFs, course handbooks, and instructor reading recommendations as weaker substitutes.

## Fit Heuristics

- A good fit usually has several core readings the user genuinely wants to read, not just a prestigious school name.
- Course/studio names should map to the user's desired direction. Do not treat a broad department match as enough.
- Shared readings across top programs are good "sample before applying" candidates.
- The absence of exciting readings is a meaningful signal: the program may still be reputable but not personally right.
- Avoid completionism. The point is to judge fit and build momentum, not to finish every listed book.

## Example Requests

- "帮我看 Goldsmiths MA Design 适不适合我，重点看 reading list。"
- "比较 RCA、UAL、Goldsmiths 的交互设计/批判设计方向，找交叉书单。"
- "我想申请教育技术硕士，用课程书单帮我判断哪些项目偏理论、哪些偏实践。"
- "给我做一个申请前的 10 篇/本 starter reading list。"
